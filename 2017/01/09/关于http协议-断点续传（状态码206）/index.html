关于http协议-断点续传（状态码206）

此文章为nginx代理服务器断点续传功能测试，并通过测试说明断点续传的实现。（当然如果是源服务器，基本上都是支持断点续传的）

请求测试：

第一次请求时nginx代理上还没有文件的缓存，所以cache miss，代理服务器会全部保存到缓存。

发起续传请求：

客户端再次发起续传请求，首先要检查Cached，如果命中则正常返回206续传
